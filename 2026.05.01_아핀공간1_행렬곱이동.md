## 핵심 개념
- 아핀공간을 사용해 행렬 곱으로 물체의 이동 기능을 구현
> 1. 행렬 덧셈으로 이동 기능을 구현할수는 있으나, 계층구조가 복잡해 질수록 관리 불가능
> 2. GPU는 대량 행령 곱셉에 최적화된 하드웨어이기 때문에 행렬 곱셈으로 한번에 처리하도록 하는것이 훨씬 효율적


## 수학적 개념
- 아핀 공간 : 점과 벡터가 공존하며 이동이 가능해진 부분 공간
> 일반적인 행렬로는 이동에 대응하는 선형 변환을 구현 할수 없음. 그래서 아핀 공간을 사용해 행렬 곱으로 이동기능을 구현
- 이동 변환행렬 : 마지막 차원 값이 1이라는 특정 조건을 가지는 전단 변환을 활용하여 행렬로 이동 기능을 구현
> $T =\begin{bmatrix}1 & 0 & a \\ 0 & 1 & b \\ 0 & 0 & 1\end{bmatrix}$  
> 이렇게 벡터공간 이동을 위해 마지막 차원 값을 1로 한정한 부분 공간을 **아핀공간** 이라고 한다.
- 아핀 변환 : 아핀 공간을 사용하기 위해 한 차원을 높여 설계한 선형 변환(동차 좌표계)
> 1. 크기 변환행렬 : $S=\begin{bmatrix}a & 0 & 0 \\ 0 & b & 0 \\ 0 & 0 & 1\end{bmatrix}\begin{bmatrix}x  \\ y \\ 1 \end{bmatrix}=\begin{bmatrix}ax  \\ by \\ 1 \end{bmatrix}\newline$<br>  
> 2. 각도 변환행렬 : $R=\begin{bmatrix}coS\theta & -sin\theta & 0 \\ sin\theta & cos\theta & 0 \\ 0 & 0 & 1\end{bmatrix}= \begin{bmatrix}xcos\theta-ysin\theta  \\ xsin\theta + ycos\theta \\ 1 \end{bmatrix}\newline$  
> - 이들의 계산 결과 역시 마지막 차원 값이 1이 되어 해당 연산은 **아핀 공간**에 갇혀 있음을 확인할수 있다.


## 게임 적용
- 게임속 캐릭터나 오브젝트의 기본 요소인 이동,크기조절,회전을 전부 행렬곱으로 구현하여 벡터 연산을 사용하는 것보다 더 효율적으로 구현

 ## 코드예제
 ```C++
 // 렌더링 로직을 담당하는 함수
void SoftRenderer::Render2D()
{
	// 렌더링 로직에서 사용하는 모듈 내 주요 레퍼런스
	auto& r = GetRenderer();
	const auto& g = Get2DGameEngine();

	// 배경에 격자 그리기
	DrawGizmo2D();

	// 렌더링 로직의 로컬 변수
	float rad = 0.f;
	static float increment = 0.001f;
	static std::vector<Vector2> hearts;
	HSVColor hsv(0.f, 1.f, 0.85f);

	// 하트를 구성하는 점 생성
	if (hearts.empty())
	{
		for (rad = 0.f; rad < Math::TwoPI; rad += increment)
		{
			float sin = sinf(rad);
			float cos = cosf(rad);
			float cos2 = cosf(2 * rad);
			float cos3 = cosf(3 * rad);
			float cos4 = cosf(4 * rad);
			float x = 16.f * sin * sin * sin;
			float y = 13 * cos - 5 * cos2 - 2 * cos3 - cos4;
			hearts.push_back(Vector2(x, y));
		}
	}

	// 아핀 변환행렬 (크기)
	Vector3 sBasis1(currentScale, 0.f, 0.f);
	Vector3 sBasis2(0.f, currentScale, 0.f);
	Vector3 sBasis3 = Vector3::UnitZ;
	Matrix3x3 sMatrix(sBasis1, sBasis2, sBasis3);

	// 아핀 변환행렬 (회전)
	float sin = 0.f;
	float cos = 0.f;
	Math::GetSinCos(sin, cos, currentDegree);
	Vector3 rBasis1(cos, sin, 0.f);
	Vector3 rBasis2(-sin, cos, 0.f);
	Vector3 rBasis3 = Vector3::UnitZ;
	Matrix3x3 rMatrix(rBasis1, rBasis2, rBasis3);

	// 아핀 변환행렬 (이동)
	Vector3 tBasis1 = Vector3::UnitX;
	Vector3 tBasis2 = Vector3::UnitY;
	Vector3 tBasis3(currentPosition.X, currentPosition.Y, 1.f);
	Matrix3x3 tMatrix(tBasis1, tBasis2, tBasis3);

	// 모든 아핀 변환을 곱한 합성 행렬. 크기-회전-이동 순으로 적용
    // 만약 이동(T)을 먼저 하고 회전(R)을 하면, 물체가 원점 기준으로 크게 원을 그리며 돌게 됩니다. 
    // 물체 자체를 제자리에서 돌리고 싶다면 반드시 이동 전에 회전을 마쳐야 합니다.
	Matrix3x3 finalMatrix = tMatrix * rMatrix * sMatrix;
     
	// 각 값을 초기화한 후 동일하게 증가시키면서 색상 값을 지정
	rad = 0.f;
	for (auto const& v : hearts)
	{
		// 곱셉을 위해 하트를 구성하는 벡터를 3차원으로 변경
		Vector3 newV(v.X, v.Y, 1.f);

		// 아핀 변환을 적용하고 마지막 차원의 값을 제거하고 사용
		Vector3 finalV = finalMatrix * newV;

		hsv.H = rad / Math::TwoPI;
		r.DrawPoint(finalV.ToVector2(), hsv.ToLinearColor());
		rad += increment;
 ```
