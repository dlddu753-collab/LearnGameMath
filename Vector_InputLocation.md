# [2026.04.27] 입력으로 벡터의 위치 변경

## 핵심 개념
- 벡터의 연산을 사용해 입력에 따라 점이 움직이도록 구현
- 델타 타임 : 프레임 간의 시간간격을 나타내는 가변적인 수치
> 이동 공식에 곱함으로써 하드웨어의 성능(FPS)과 관계없이 항상 일정한 속도로 물체를 이동시키는 '프레임 독립성'을 확보할 수 있다."
>> ex) 100.f라는 고정된 속도로 이동시킨다면, 성능 좋은 컴퓨터에서는 1초에 이 함수가 144번 호출되어 14,400만큼 이동하고, 똥컴(?)에서는 30번 호출되어 3,000만큼만 이동하게 됩니다.


## 수학적 개념
- 항등원 : 연산결과가 언제나 주어진 원소가 되는 특별한 원소.
> 덧셈은 0, 곱셉은 1 ex)  1 + 0 = 1 , 1 * 1 = 1 <- 덧셈 = 0, 곱셈 = 1 항등원
- 역원 : 연산결과가 언제난 항등원이 되는 원소
> 덧셈은 반대수, 곱셈은 역수 ex) 1 + (-1) = 0, 2 * 1/2 = 1 <- (-1), 1/2이 역원
- 곱집합 : 두 집합의 원소를 순서쌍으로 묶은 원소의 집합.
 > 곱집합의 두 집합 A,B의 요소를 수직으로 배치하여 배열의 형태를 나타냄.
- 데카르트 좌표계 : 곱집합을 사용해 직선의 수 집합을 수직으로 배치해 평면을 표기하는 방식.
- 좌표 : 데카르트 좌표계의 원소를 곱집합과 동일하게 순서쌍으로 표현한것. `(x,y)`

- 벡터 공간 : 곱집합으로 형성된 집합을 공리적 집합론 관점에서 규정한 것. `V`

- 벡터 : 벡터 공간의 원소를 나타냄
> 벡터 = 곱집합 = 좌표 = 체 집잡의 원소
- 스칼라 : 체의 구조를 가지는 집합의 원소
> 체 집합 : 좌푯값으로 사용하는 X,Y 값
- 벡터의 합 : (1, 2) + (3, 1) = (4, 3)
> 결합법칙,교환법칙,항등원,역원
- 벡터의 곱 : 2 * (1, 2) = (2, 4)
> 호환성,항등원,분배법칙


## 게임 적용 
- 벡터 공간에서 캐릭터의 시작위치를 좌표값을 지정하여 설정.
 ## 코드예제
```c++
// 이동할 위치 좌표값을 가진 벡터 객체 생성.
Vector2 currentPosition(100.f,100.f);

// 게임 로직을 담당하는 함수
void SoftRenderer::Update2D(float InDeltaSeconds)
{
	// 게임 엔진 모듈 접근
	auto& g = Get2DGameEngine();
    // 게임 엔진 모듈에서 사용자 입력값을 읽을수 있도록 입력 관리자를 레퍼런스를 읽기 전용으로 가져옴
	const InputManager& input = g.GetInputManager();

	// 1초에 몇 픽셀을 움직일 것인가
	static float moveSpeed = 100.f;
    // X축Y축 입력을 결합해 입력 벡터를 생성
	Vector2 inputVector = Vector2(input.GetAxis(InputAxis::XAxis), input.GetAxis(InputAxis::YAxis));
    // 컴퓨터 성능에 따른 이동거리를 보정해 주기 위하여 InDeltaSeconds값 곱연산
	Vector2 deltaPosition = inputVector * moveSpeed * InDeltaSeconds;

	currentPosition += deltaPosition;
}

```
